# TM-1 Phase 3b: MO v3 — signatures, table blobs, binding records

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Extend the MO object container to version 3 so it can carry generic
TM-1 routines: per-routine signatures (tape arity + per-tape alphabet
cardinalities), per-routine table blobs (a routine's own match/dispatch
tables), table fixups (mtc/djmp operand holes), and call-site binding records
(projection + per-tape symbol maps incl. one-way pairs) — the composition
engine's input (spec §6.2, §5.2). PM-1 objects keep emitting byte-identical
v2.

**Architecture:** Spec §6.2; same version-tag pattern proven in phase 3a:
single `ObjectFile` struct, new fields defaulted/empty, `to_bytes()` emits
byte-identical v2 when no v3 data is present (`is_v2_shape()`), else v3 with
new trailing sections appended after the debug section. Readers accept
versions 1..=3. Producers/consumers of the new records are LATER phases
(assembler emit in phase 4, linker consume in phase 5) — this plan is the
codec + validation + tests only.

**Tech Stack:** Rust; no new dependencies; proptest already a dev-dep.

## Global Constraints

- **PM-1 `.pmo` byte-identity**: an `ObjectFile` without v3 data must emit
  byte-for-byte what v2 emits today (locked by a test; there are no golden
  `.pmo` files, so the lock test is the gate). `cargo test --workspace`
  green at the end of every task (golden_programs/cli_programs/opt_equivalence
  exercise the real .pmo pipeline end-to-end).
- `cargo clippy --workspace --all-targets -- -D warnings` and
  `cargo fmt --check` clean before every commit.
- Every reader verifies CRC-32 before decoding anything (v3 path included).
- `sniff()` untouched (magic-only dispatch).
- `crates/core/src/formats` carries zero architecture knowledge — arity,
  cardinalities, tape indices, and map pairs are stored verbatim; only
  structural bounds are validated (the SEMANTIC mapping rules — bijection
  completion, hole legality — are the linker's job in phase 5, not the
  format's).
- No new dependencies. Commit style: conventional with scope. NEVER add any
  Claude/AI attribution footer.
- Commits require the maintainer's explicit go-ahead in the executing
  session; if not granted, stop at the commit step and ask.
- Structural caps enforced by `from_bytes` (mirroring the machine contracts):
  routine arity 1..=16; every cardinality ≥ 1; `caller_tape` < 16; map-pair
  flags may set only bit 0 (one-way).

```rust
pub const OBJECT_FORMAT_VERSION_V2: u16 = 2;
pub const OBJECT_FORMAT_VERSION_V3: u16 = 3;

/// A generic routine's signature (spec §6.2): its virtual tape arity and
/// per-tape alphabet cardinality. Parallel to `blobs` when present.
#[derive(Debug, Clone, PartialEq, Eq)]
pub struct RoutineSig {
    pub arity: u8,                    // 1..=16
    pub cardinalities: Vec<u32>,      // len == arity, each >= 1
}

/// An mtc/djmp operand hole: the u32 at `offset` inside `blob`'s code is
/// an offset into that blob's OWN table blob; the linker rebases it into
/// the final table section.
#[derive(Debug, Clone, Copy, PartialEq, Eq)]
pub struct TableFixup {
    pub blob: u32,
    pub offset: u32,
    pub table_offset: u32,
}

/// One caller-symbol → callee-symbol map entry. `one_way` = read-only
/// (collapse allowed, excluded from write-back; spec §5 `=>` pairs).
#[derive(Debug, Clone, Copy, PartialEq, Eq)]
pub struct MapPair {
    pub src: u32,
    pub dst: u32,
    pub one_way: bool,
}

/// One virtual-tape binding at a call site: which caller tape feeds this
/// callee tape, and the symbol map between their alphabets.
#[derive(Debug, Clone, PartialEq, Eq)]
pub struct TapeBinding {
    pub caller_tape: u8,              // < 16
    pub pairs: Vec<MapPair>,
}

/// A declarative bound call site (`call name [binding]` in .tma): the
/// composition engine's input. `offset` marks the call operand hole in
/// `blob`, like a Relocation; `binding[k]` binds the callee's virtual
/// tape k.
#[derive(Debug, Clone, PartialEq, Eq)]
pub struct BoundCall {
    pub blob: u32,
    pub offset: u32,
    pub symbol: u32,
    pub binding: Vec<TapeBinding>,
}
```

  blobs), `pub table_fixups: Vec<TableFixup>`, `pub bound_calls: Vec<BoundCall>`.
- `fn is_v2_shape(&self) -> bool` = all four are None/None/empty/empty.

### Task 2: v3 emit + parse — signatures and table blobs

**Files:**
- Modify: `crates/core/src/formats/object.rs`

**MO v3 wire layout** (normative; extends the v2 body — everything through
the debug section is IDENTICAL to v2 except the version field = 3; then):

```
new flags bits:  FLAG_HAS_SIGNATURES = 0b0000_0010
                 FLAG_HAS_TABLES     = 0b0000_0100
after the debug section (or after relocations when no debug):
  if FLAG_HAS_SIGNATURES: per blob (blob_count entries):
      arity u8 ; cardinalities: arity × u32 LE
  if FLAG_HAS_TABLES: per blob:
      table_len u32 LE ; table bytes
  table_fixup_count u32 LE ; per fixup: blob u32, offset u32, table_offset u32
  bound_call_count u32 LE ; per call:
      blob u32, offset u32, symbol u32, tape_count u8
      per tape: caller_tape u8, pair_count u16 LE
                per pair: src u32, dst u32, flags u8 (bit0 = one_way)
```

(The fixup/bound-call sections are unconditional in v3 — a zero count is two
0-bytes… four bytes each; fine.)

## Self-review notes (spec → plan coverage)

- §6.2 in full: signature (arity M, per-tape cardinality) → RoutineSig;
  "code + own tables" → table_blobs + TableFixup; call-site records
  (relative bindings: projection + per-tape maps) → BoundCall/TapeBinding/
  MapPair with the §5 one-way bit; "usual exports/imports/relocations"
  unchanged; "new record kinds = MO version bump" → v3.
- The §5.2 composition engine's input shape (per-callee-virtual-tape
  `(caller_tape, pairs)`) is exactly `Vec<TapeBinding>` indexed by callee
  tape position.
- Byte-identity follows the proven 3a pattern (equal-valued rename +
  verbatim v2 body + lock test); .pmo has no golden files, so the lock test
  carries the guarantee, with the full pipeline suites as the end-to-end
  check.
- Deliberately absent: exit vectors in bound calls (frames-profile exit
  vectors are LINK-time artifacts in the MX table section, not object-level
  — spec §5.5), graph fragments (compile-time, never reach objects), and
  semantic map validation (linker, phase 5).
