# TM-1 arc — phase 6b: the optimizer and the stdlib twins

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** `-O1` exists and is proven — the TM optimizer's pass pipeline (ported motion passes + the new TM passes + `outline` default-off) transforms the per-world state-graph IR under a 2×3 equivalence matrix (`-O0`/`-O1` × mono/frames/hybrid), and the stdlib twins `std::binaryNumbers` / `std::binaryNumbersBare` ship as embedded source with derivation-first goldens.

**Architecture:** the pmc optimizer architecture transposed to state graphs (facts: .superpowers/sdd/phase6b-facts.md §1-§4 — the implementers' detailed reference): PROGRAM_PIPELINE (`inline`, `outline`) at round start + per-world PIPELINE (`jump_threading`, `tail_call`, `tail_merge`, `dce`, `dead_rows`, `dispatch_select`) in a fixpoint with MAX_ROUNDS 10; `-O0` returns immediately (bit-identity); per-pass debug `validate_world`; the brk barrier per-pass. The stdlib mirrors pmc's mechanism verbatim (include_str! + OnceLock at -O1-stripped + reachability-linked + `--nostdlib`), content per spec §12 from the js inventories (facts §5).

**Tech stack:** Rust 2024; no new dependencies.

**Plan-level adjudications (RECORD in reports; the second is flagged to the maintainer):**
1. **TM_IR_VERSION 1 → 2**: `IrTransition::TailCall { target: String }` (tail_call's output; codegen emits `jmp @target`) and `IrState.dispatch: IrDispatch { Table (serde default) | Branch }` (dispatch_select's hint) are IR-shape additions. Internal + unreleased, but the version contract moves with ANY shape change. The release version block will show TM IR 2.
2. **`outline` ships the single-junction call/ret form ONLY** (mode-independent objects). The spec's §11.4 multi-exit exit-vector variant would make the compiler emit raw `.frame`/`call.m`/`retx` — a frames-profile-LOCKED object that `--call-mech=mono` refuses, contradicting §11.1's "objects are mode-independent". Deferred with the trigger recorded (per-mode outlining needs either a mode-aware compile flag or linker-side outlining — a design decision above this plan's pay grade). Single-junction outlining satisfies the §11.4 contract shape (default OFF, `--foutline`, brk refusal, thresholds disjoint from inline's).
3. **`dispatch_select` scope** (the jm/jnm arithmetic): `jm`/`jnm` test MR≠0 only, so branch lowering can express exactly "matched vs not" — the pass targets states with 2 rows where the LAST is an all-wildcard catch-all (row 1 exact/partial + catch-all): the match table keeps ONLY row 1, the dispatch table is dropped, codegen emits `rd; mtc T; jm <row1>; <catch-all body>` (fall-through). 1-row-no-catch-all states KEEP the table path (MR=0 must trap NoTransition via djmp — no branch equivalent exists). This is the "tables primary + special cases" reading.
4. **`inline` scope**: bindingless same-unit calls only (identity world-compat); binding-aware inlining is the graft machinery and stays out of the pass (a recorded trigger: measured wins on the live stdlib).

## Global Constraints

1. **-O0 bit-identity**: `optimize` at O0 returns immediately, never touching ir/snapshots; the equivalence harness locks `-O1 --fno-<every-pass>` object == `-O0` object (the pmc do-no-harm floor test transposed).
2. **Order contract**: `tail_call` BEFORE `tail_merge` in PIPELINE (return-chaining destroys the precondition — pmc's stated contract carries).
3. **brk barrier**: every MOTION pass refuses across an un-stripped `debugger` rule (`IrRule.debugger`); `outline` refuses any group containing one; the harness asserts o0==o1 observables across a debugger-bearing program. The optimizer runs before codegen's strip (facts §4) — the barrier always holds.
4. **Pass invariants**: after every pass, debug builds assert `validate_world` per world (dense ids — a deleting/merging pass renumbers and retargets EVERY `Goto`/`IrThen::Goto`; facts §3 lists the 9 invariants; codegen's entry-first + fall-through assumptions are facts §4).
5. **Equivalence contract** (the pmc wording transposed): passes preserve final tapes, termination kind, and trap KIND on the same seeds across the FULL 2×3 matrix (`-O0`/`-O1` × mono/frames/hybrid); step counts and resource-limit outcomes may change; no motion across an un-stripped brk.
6. **--foutline plumbing**: `OptOptions` gains `outline: bool` (default false); the CLI flag `--foutline` sets it; `outline` appears in `pass_names()` (so `--emit-ir=after:outline` resolves and `--fno-outline` parses as a validated no-op — documented).
7. **Stdlib mechanism** = pmc verbatim (facts §6): `stdlib/mod.rs` with `SOURCE`/`object()` (OnceLock, `-O1` + strip), `tmt link --nostdlib`, reachability-linked; a drift test asserts the namespace exports == the compiled object's exported symbols.
8. **Stdlib content** per spec §12 + the js inventories (facts §5 — the port's normative contracts incl. head positions and the four edge cases): two namespaces, each `export alphabet` + behavior-as-`export graph`s + one-line `export routine` facades (`entry graft g(…, done = return);`); delimited ports all ten routines (incl. BOTH `minusOne` — the ~(~x+1) composition, as a routine calling facades in sequence since graphs cannot contain calls — and `minusOneFast`); bare ports four. The `goToNumbersStart` js-internal-name typo is NOT carried (our state names are internal anyway). Cross-representation composition (delimited `invertNumber` over `bare::invertNumber` with `'^' => '_', '$' => '_'`) is ATTEMPTED as the showcase; if blocked by anything structural, fall back to independent implementation and record why.
9. **Goldens derivation-first** (per-routine: derive expected tapes from the js contracts by hand — never from runs); the A-example goldens stay green untouched.
10. PM-1 byte-identity; core untouched EXCEPT nothing (this phase is all crates/turing-machine — any core need is a finding, not a change); no new deps; thin renderer; conventional commits, NO attribution footers; durable citations only (no docs pages this phase — optimizer/stdlib pages are phase 8; prose in code); no `§N`/`GC{n}` notation in code comments (write briefs accordingly — sweep before merge regardless).
